# Research Roadmap for FlashVTG-GMR

## Scope and Provenance

This roadmap is based on the analyses in
`/home/guoxiangyu/HieA2G_GMR/error_analysis_tasks/`. Those analyses were
generated from the original FlashVTG-GMR method used for the GMR paper, not
from the current HieA2M implementation or the A6 reproduction experiments.
The original run below is the baseline source of truth; no result from the
current project should be substituted for it.

The objective is to improve both positive-query localization (mAP) and
end-to-end set prediction (G-mIoU), without obtaining G-mIoU gains by simply
rejecting more queries.

## Main Diagnosis

The candidate pool is substantially better than its final ordering and set
prediction:

- Post-NMS proposal coverage is 93.23% at IoU 0.5, while top-5 recall is only
  62.98%.
- Proposal score and IoU have weak rank correlation (Spearman 0.295).
- NMS improves mAP from 23.88 to 25.72 and mR+@5 from 16.69 to 21.57, but
  improves G-mIoU@5 by only 0.20 at the official 0.4 gate.
- Oracle reranking raises mAP from about 25.72 to 58.57.
- At the diagnostic 0.5 gate, oracle existence decisions raise G-mIoU@3 from
  33.36 to 54.65.
- Oracle unique-moment selection plus oracle existence reaches about 87.33
  G-mIoU@3 at the same diagnostic gate.

These are diagnostic ceilings, not reportable method results. Since 47.49% of
Standard test queries are null, an always-empty predictor obtains 47.49
G-mIoU. Every experiment must therefore report rejection, positive-query
localization, and end-to-end metrics together.

## HieA2G Transfer Boundary

Transfer hierarchical semantic alignment: word-to-proposal, phrase-to-moment,
and query-to-video reasoning can help distinguish action, actor/team, and
global context. Count-aware representation learning may also be retained if
the target is the number of latent temporal modes.

Do not directly transfer `predicted count -> proposal Top-K`. Object candidates
in GREC represent relatively discrete entities, whereas FlashVTG proposals are
many overlapping observations of the same temporal event. Count should be a
soft prior over event modes, never a hard quota that deletes the localization
list.

## Ranked Research Directions

### 1. Compositional Constraint-Aware Proposal Verification

Factor each query into action and actor/team constraints. For every proposal,
predict action satisfaction, actor satisfaction, and expected temporal IoU,
then learn a calibrated ranking score. Train with within-video hard negatives:
correct action/wrong team, correct team/wrong action, and high-scoring temporal
distractors. Estimate existence from whether any proposal satisfies the full
constraint conjunction.

This is the highest-confidence joint direction: better proposal ordering
improves mAP, while complete-constraint verification can reject partial
semantic matches. The principal risk is that existing frozen features may not
resolve player or team identity. Validate factorized versus monolithic scoring
within the WorldCup subset to remove the current dataset-source confound.

### 2. Quality-Aware Temporal-Mode Set Decoder

Treat dense proposals as samples from latent event modes. Learn proposal
quality, pairwise same-event affinity, soft mode assignments, and one
representative per mode. Train mode slots with Hungarian or unbalanced-OT
matching and a null dustbin. Use cardinality only as a regularizer on active
modes.

At inference, place one quality-ranked representative from each mode first,
then retain other nonduplicate proposals in the tail up to the official limit
of ten. This front-loads a useful set for G-mIoU while preserving mAP recall.
Validate by moment count and inter-moment gap because nearby events may be
incorrectly merged.

### 3. Continuous Boundary Distribution and Quality Refinement

First disable the current two-second boundary rounding. Reconstructed
continuous coordinates already improve mAP by about 0.62, so this is a required
baseline correction rather than a research contribution.

Then predict continuous start/end distributions, boundary uncertainty, and
expected IoU using fine temporal features. Use expected quality in reranking.
This targets the 38 of 48 coverage failures that are near misses and the weak
82.7% coverage for moments no longer than three seconds. Evaluate coordinate
refinement and score refinement separately before combining them.

### 4. Proposal-Set-Aware Existence Calibration

Replace global mean pooling with a DeepSets or Set Transformer head over
proposal qualities, score histograms, temporal dispersion, inferred modes, and
compositional satisfaction. Train on balanced, within-source positives and
negatives, then calibrate probabilities on validation data only.

This direction primarily improves rejection and G-mIoU, so it should support
Direction 1 or 2 rather than serve as the only contribution. Use gradient
stopping or separate optimization if existence training damages proposal
ranking.

### 5. Expected Set-Utility Decoding

Learn a query-adaptive policy that orders temporal modes to maximize expected
set IoU under boundary and existence uncertainty. Preserve the full ranked
proposal list and expose a continuous existence probability; do not perform
AMC-style destructive gating.

This directly generalizes query-adaptive NMS, soft gating, and count-aware
reranking. It is higher risk because a metric surrogate may overfit, so compare
it against quality-only reranking and mode decoding with identical candidates.

## Experimental Program

1. Establish a clean baseline from scratch through the original repository's
   `scripts/train_flash_vtg_gmr.sh`, with batch size 8, no resume checkpoint,
   one frozen code snapshot, and explicit environment paths. Run three seeds
   and select checkpoints using validation data only.
2. Treat the supplied mixed-batch checkpoint and its 25.72 test mAP as a locked
   historical reference, not as the sole experimental control for new methods.
3. Archive hashes for the historical checkpoint, code snapshot, and T1-T7 test
   artifacts; reproduce their locked metrics before changes.
4. Disable boundary rounding and add proposal identifiers around NMS.
5. Repeat oracle decomposition on `data/label/Standard/val.jsonl`; do not use
   existing test diagnostics for architecture or threshold selection.
6. Implement and train each direction independently with the same backbone,
   data, schedule, and three random seeds.
7. Advance a direction when it improves validation mAP by at least 1.0 and
   G-mIoU@3 by at least 2.0, with paired-bootstrap confidence intervals above
   zero and no more than a 0.5 drop in mR+@5.
8. Combine only independently successful components, then run Standard test
   once. Repeat the baseline and selected model on the Full split as a separate
   Soccer-GMR scaling experiment.

## Evaluation and Ablations

Always report AUROC, Rej-F1, accuracy, false-positive and false-negative rates,
mAP, mR+@5, positive-query mIoU, and G-mIoU@1/3/5. Include the always-empty
baseline and stratify results by null/single/multi-target query, action type,
duration, moment count, inter-moment gap, and dataset source.

Required ablations are semantic relevance versus IoU quality, action versus
actor verification, quality-only versus mode-aware selection, fixed NMS versus
learned affinity, continuous coordinates versus boundary refinement, and
independent versus shared existence training.

## Output Contract and Exclusions

Retain the official output fields: ordered continuous
`pred_relevant_windows` and calibrated `pred_exist_score`. Neither existence
nor predicted cardinality may delete the stored proposal list. Do not use
`match_info` or team metadata at inference.

Deprioritize forced scale balancing, temporal-position debiasing, score-entropy
regularization, fixed Soft-NMS, and additional proposal generation. The current
diagnostics do not identify these as primary bottlenecks.
